体验地址：[https://cpti.cc)

CPTI 的定位很直白：**移动端优先**的恋爱向人格测试 **静态产品原型**——不依赖后端服务，打开 HTML 就能跑完整流程；适合快速验证文案、题型和结果呈现，也方便丢到 GitHub Pages 或任意静态托管上对外演示。

---

## 一、为什么要单独拆页

当前版本把体验拆成 **`index.html`（首页）**、**`quiz.html`（测试页）**、**`result.html`（结果页）** 三个独立入口，而不是单页一把梭：

- **首屏更轻**：首访只加载首页所需资源，减少「一上来就拖整套题库」的压迫感。
- **心智分段**：「了解产品 → 认真答题 → 看结果分享」三条链路各自独立，利于以后做深链接或运营投放。
- **静态友好**：每页仍是纯 HTML + JS + CSS，部署模型简单。

## 二、核心能力一览（摘自项目 README）

| 维度 | 说明 |
|------|------|
| **题型** | 三选一、四选一、聊天场景题等；共 **34 道**恋爱向题目（含场景题与少量「恶搞」调节题） |
| **结果** | **双输出**：① 你是哪种恋爱人格；② 同一套人格谱系里，哪一型更适合「接住你」 |
| **人格库** | **20 个**共享恋爱人格类型，文案与计分集中在 `data.js` |
| **MBTI** | 可选填写，作为 **轻度辅助权重** 参与计算（不是硬绑定 16 型刻板标签） |
| **结果页** | 支持 **一键复制**结果文案，方便用户丢到备忘录或社交平台 |

![测试页：瀑布流短题卡示意](assets/uploads/2026/05/cptijietu2.png)

测试页采用 **瀑布流短题卡**：在手机上单手滑动、点选的节奏更接近「刷内容」而不是「填问卷」，有利于做完 34 题。

## 三、技术栈与文件结构

实现上就是经典的 **无构建链静态站**：

- **`styles.css`**：三页共享的视觉与排版。
- **`data.js`**：题库、人格类型定义与 **计分 / 归一化逻辑**（单一数据源，改题改文案主要动这里）。
- **`quiz.js`**：测试页交互（题卡切换、答题记录、与结果衔接）。
- **`result.js`**：结果页渲染与复制等能力。

仓库里也列了后续方向，例如把题库抽成独立 JSON 方便运营维护、加强 MBTI 映射、增加分享海报与埋点等——都很适合在「验证完原型」之后按优先级迭代。

## 四、本地预览与部署

- **本地**：用浏览器直接打开 `index.html`，从首页即可进入测试页与结果页（与 README 描述一致）。
- **线上**：仓库已开启 **GitHub Pages**，可配合自定义域名做轻量演示环境；静态资源路径保持相对引用即可。

若你希望把 CPTI 嵌进现有博客导航，只需像其他外链工具一样在 `config.js` 的导航里加一项指向 Pages 地址即可。

## 五、小结

CPTI 用很小的静态面，把一个「恋爱人格测试」产品从 **引流 → 答题 → 结果消费** 的闭环跑通；**双结果 + MBTI 轻辅助** 让它不只是娱乐测试，也留了一点严肃关系讨论的余地。若你对题型统计、文案 A/B 或结果可视化有想法，欢迎在 [Issues](https://github.com/flymysql/cpti/issues) 里聊，或直接 Fork 改 `data.js` 做自己的变体。
